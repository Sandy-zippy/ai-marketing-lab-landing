03b / Start here

Never coded? You're not behind. You're early.

AI agents aren't sci-fi. They're tooling. If you can describe what you want, you can build it. Here is what each tool is actually for, in plain English. Pick one. Start at your pace. The community helps with the rest.

01 / The architect
Claude Code
Plans before it builds.

Use it when you have a fuzzy idea and need a real plan. Claude Code is good at thinking across many files, deep reasoning, and designing a system before a single line of code gets written. Talk to it like a senior engineer who actually reads the whole brief.

Ask Claude what to build, before you build it.
02 / The fast hands
Codex · ChatGPT
Turns a plan into a working thing.

Once the plan exists, Codex and ChatGPT are the fast hands that ship it. Daily ops, content, quick scripts, small workflows. Type the request, get the working output, move on. This is where most of your day actually happens.

Ship a workflow before your coffee gets cold.
03 / The cron worker
Antigravity · Gemini Flash
Runs while you sleep.

For the boring, high-volume work that should never touch your hands. Antigravity and Gemini Flash are cheap enough to run all day, schedule overnight, and chew through tasks in the background. Think: tagging, sorting, summarising, polling.

Your overnight assistant. Cheaper than coffee.
04 / The wires
n8n · Composio
Connects everything you already use.

The plumbing. n8n and Composio wire your tools together so Gmail, Slack, GHL, Stripe, your CRM, and the AI models above can finally talk. No more copy-paste. No more spreadsheet babysitting. Set the wire once, let it run forever.

Stop copy-pasting. Let your tools talk.

Start with one tool, one outcome. The community helps you string them together. No prerequisites except the willingness to build.

04 / Inside Hermes

Autonomous. Self-improving. Multi-LLM. Always on.

Hermes is the operating system behind ZippyScale. 5 agents, 5 models, cron-driven, memory-anchored. Here's how every agent ships work while I'm asleep.

01 · CRON

Every morning, the fleet wakes up.

Each agent gets its slot. D-Rose dispatches the floor at 09:00. Ironman never sleeps. The radar arm below sweeps a 24-hour day in 12 seconds. Watch each agent light up when its turn comes.

  • 5 cron jobs, 1 VPS, zero babysitting
  • Outputs delivered to Telegram by D-Rose
  • Failure events auto-escalate to Sandy via Ironman
D-ROSE · 09:00 IST CURRY · 14:00 ANTMAN · 16:00 KYRIE · 09:05 + 22:00 IRONMAN · always on · continuous monitor (no fixed cron)
02 · LLM ROUTING

Each agent picks its driver LLM by task.

Claude for the thinking. Codex for the throughput. Gemini for the volume. Hermes routes every request through a model picker so each agent runs on the LLM that's best at its job — and cheapest at its load.

  • Claude · Opus 4.7 · reasoning + architecture
  • Codex / ChatGPT · GPT-5.5 · fast code + research
  • Gemini Flash · 2.5 · async cron volume
03 · SELF-IMPROVE

Every run leaves a fingerprint.

Every run leaves a fingerprint in Mem0 + gbrain, our memory layer. Mem0 stores the persistent context. gbrain orchestrates retrieval across the fleet. Every fingerprint sharpens the next run. The system gets smarter while I sleep.

  • 5-step cycle: TRIGGER → EXECUTE → LOG → REFLECT → UPDATE
  • Mem0 + gbrain grow with every run · queryable across the fleet
  • Execution snippets anchored to context · iterating daily
04 · OPEN CLAW

Every channel I work in routes through Open Claw.

D-Rose reads first. Specialists handle what's theirs. I see only what needs me. 24+ inbound channels converge into a single inbox — humans, agents, webhooks, calendars, all triaged by the same orchestrator.

  • 12 visible · 12 more behind the scenes
  • D-Rose triages, specialists execute
  • Sandy sees only escalations + approvals
05 · TOOLS

Every agent gets 100+ APIs without writing integration code.

Composio is the tool layer. One config, every API ready. Gmail, Slack, GHL, Stripe, Notion, calendars, CRMs — agents call them like they call functions.

  • 100+ tools standardised behind one auth + schema layer
  • Agents call Gmail / Slack / GHL / Stripe / Meta / Google Ads as functions
  • No bespoke API code · no broken refresh tokens · no glue scripts

09b / Why I'm building this

14 years on the sales floor. Then AI rewrote the job description.

The Lab is not a content move. It's the room I needed and could not find.

2012 to 2020

I learned to think in revenue, not impressions.

I started in Mumbai. Cold calls, sales targets, weekly close numbers pinned to a wall. Five years running sales and marketing at a consumer brand taught me one thing: revenue is the only metric that survives a Monday review. Marketing that does not move it gets cut.

Then I ran my own D2C brand for three years. My money, my product, my P&L. Meta Ads, Google Ads, supplier calls, fulfilment, customer service. Every rupee spent had to earn its way back. That stretch is where I stopped being a marketer who talks about funnels and became an operator who reads them.

2020 to 2025

ZippyScale, and the moment manual broke.

I started ZippyScale to build the one thing every agency in this country lies about: real attribution. Every rupee traced, every channel scored, every client decision defended with data. That promise carried us across 25+ brands and a chunk of tracked revenue I am proud of.

Then it stopped scaling cleanly. Reports took hours. Outreach took weeks. Creative briefs sat in drafts. The agency hit the wall every operator hits, when the only growth lever left is hire more people, and the math stops working. I was not going to run a body shop. I needed leverage.

2026, the awakening

I built Hermes. Then I watched it run.

Hermes is the five-agent fleet that runs ZippyScale. It lives on a small VPS, talks to Claude Code, Codex, ChatGPT, Gemini and Antigravity, remembers what we said last week, and ships work on a cron schedule. It is not a chatbot. It is not a GPT. It is operators stitched together from the best models on the market, pointed at real revenue jobs.

The first week it ran end-to-end, my role inside the agency changed. I stopped being the bottleneck and started being the architect. That is the shift I want every operator on this page to feel. Not "AI helped me draft a tweet." "AI shipped the work while I was at dinner."

Why community, why now

Every operator I know is six months behind a moving target.

A course goes stale the day it ships. The frontier moved this morning. The operator class, heads of growth, founders running their own marketing, agency owners with 5 to 30 clients, does not need another 12-hour playlist. They need a room that builds in front of them every week, with a working fleet on the table.

That is the Lab. We open the hood on Hermes. You watch the agents run, fork the pieces you want, and ship your own. Every build comes with the measurement plan attached, so the agent shows up in your revenue report or it gets killed. No theory. No tactics-of-the-week. One operator, one room, agents in production.
